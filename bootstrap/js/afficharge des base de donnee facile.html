Dans Symfony, afin d'interagir avec une base de données, on utilise un ORM, Object-Relating Mapping. L’ORM, par défaut dans Symfony, est Doctrine. Doctrine crée des entités qui permettent de manipuler la base de données à travers des objets. Ces entités correspondent aux tables de notre base de données. Elles sont utilisés dans des contrôleurs pour faire des requêtes qui donnent un résultat affiché dans la vue. Cet article fera référence à Symfony3. Nous allons tout d’abord découvrir ensemble comment créer une base de données avec Doctrine.

Si nous rencontrons des problèmes lors de la régénération des entités après avoir exécuté les 3 commandes ci-dessus, il faut vider notre cache et relancer PHP :

  • php bin/console cache:clear = commande sous Symfony3

  • php app/console cache:clear = commande sous Symfony2

Pour relancer PHP, il faut se connecter en tant que root sur notre machine de dev :

  • service php5-fpm restart

find($id)

La méthode find($id) récupère tout simplement l'entité correspondant à l'id $id.

findAll()

La méthode findAll() retourne tous les objets d’une entité contenue dans la base de données. Le format du retour est un simple Array, que vous pouvez parcourir (avec un foreach par exemple) pour utiliser les objets qu'il contient.

QueryBuilder

QueryBuilder permet de faire des requêtes plus développées comme par exemple :

  • c correspond à l’entité renseignée par la variable $repository.

  • Ce qui suit le . après “c” est le nom des champs dans l’entité. Attention, le nom doit être écrit exactement comme c’est écrit dans l’entité et non dans la base de données !

  • On peut mettre des conditions dans la requête via “where” :

    Dans la condition, on renseigne le champ où on veut faire la condition ainsi que la valeur dont celui-ci doit prendre. La condition se fait via deux lignes :

    • where('c.slug = :slug')

    • setParameter('slug', test)

  • :slug est la valeur que doit avoir le champ slug contenu dans l’entité c.

  • slug qui est dans le paramètre (setParameter) correspond à :slug et donc doit être écrit identiquement sans les “:”. test est la valeur.

Donc dans ce cas, le slug doit être égal à test.
